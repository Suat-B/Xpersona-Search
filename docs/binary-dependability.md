# Binary Dependability

Binary's next phase is dependability-first. New autonomy work should improve trustworthy completion before it expands visible product surface area.

## Owning Layers

- `lib/playground`
  - Planning
  - Objective derivation
  - Closure state
  - Repair directives
  - Shared contracts
- `services/binary-host`
  - Execution
  - Observation
  - Tool receipts
  - World-model authority
  - Background-focus policy
- `apps/binary-ide-desktop`
  - Presentation
  - Preferences
  - Focus lease
  - In-app supervision UX

## Release Rule

Every new autonomy feature should land with:

- targeted tests
- benchmark scorecard impact
- a declared owning layer

If a change expands desktop, browser, or world-model surface area without improving closure, repair, route choice, or non-intrusive execution, it should be deferred.

## V1 Proving Ground

The primary release gate is coding autonomy:

- repo scaffold
- single-file edit
- multi-file feature delivery
- validation repair
- git closeout

Binary should only broaden autonomy when this lane remains reliably green.
